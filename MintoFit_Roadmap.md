# MintoFit — Roadmap

## Концепція

Особистий застосунок для тренажерного залу. Тільки для себе. Без комерції.  
Стек: **React + Vite + Tailwind CSS + Supabase + Vercel**

---

## Дизайн і UX

- Темна тема як основна
- Один акцентний колір (вибрати на етапі дизайну)
- Великий читабельний шрифт
- Мобільний перший — застосунок відкривається з телефону в залі
- Три вкладки внизу: 🏋️ Тренування / 📊 Прогрес / ⚙️ Програми

## Фази розробки

### Фаза 1 — База даних (Supabase)

- [x] Створити проєкт MintoFit в Supabase
- [x] Налаштувати Supabase Auth — Magic Link (вхід через email, без паролю)
- [x] Таблиця `mf_programs` — програми тренувань
  - `id, name, type (основна/додаткова), color, emoji, created_at`
- [x] Таблиця `mf_exercises` — вправи і тренажери
  - `id, name, machine_photo_url, youtube_url, muscle_group, personal_note, created_at`
  - `personal_note` — нотатка про налаштування тренажера (висота сидіння, позиція валиків тощо), підтягується одразу з фото
- [x] Таблиця `mf_program_exercises` — які вправи в якій програмі
  - `id, program_id, exercise_id, order, default_sets, default_reps, default_weight`
- [x] Таблиця `mf_alternative_exercises` — замінники тренажерів
  - `id, exercise_id, alternative_exercise_id`
- [x] Таблиця `mf_workouts` — записи тренувань
  - `id, user_id, program_id, started_at, finished_at, duration_minutes, intensity, calories_burned, created_at`
- [x] Таблиця `mf_workout_sets` — кожен підхід
  - `id, workout_id, exercise_id, set_number, weight, reps, completed, created_at`
- [x] Таблиця `mf_body_stats` — вага і заміри тіла в одному записі
  - `id, user_id, recorded_at, weight_kg, chest, waist, hips, left_thigh, right_thigh, left_calf, right_calf, left_arm, right_arm, wrist`
- [x] Налаштувати RLS policies для всіх таблиць з user_id

- [x] Сповіщення після таймера — вібрація + короткий звук (щоб не дивитись в екран між підходами)
- [x] Таймер тривалості тренування (загальний)
- [x] Показ результату минулого тренування (яка була вага, скільки підходів)
- [x] По натисканню на вагу або повтори число прямо в таблиці перетворюється на поле вводу

- [x] Кнопка "+" в footer картки вправи — додати зайвий підхід по відчуттям; розміщується між "Нотатки" і "Замінити"
- [x] Меню ⋯ біля назви вправи (зараз неактивне): іконка 📷 → відкриває фото тренажера; іконка ▶️ → відкриває YouTube відео; дані беруться з `mf_exercises` (`machine_photo_url`, `youtube_url`,іконка інфо - показує розписану технологію виконування вправи - дані беруться в бд)
- [x] Замінна вправа — повний рефакторинг: показувати опис, підходи з чекбоксами, вагу, повтори; оригінальна вправа ховається (не видаляється), замінна підставляється на її місце; додати альтернативи до всіх вправ (список надасть користувач)
- [x] Блок кардіо-розминки: перший блок перед вправами кожного тренування — dropdown (Сходи / Еліпс / Бігова доріжка / Велотренажер), поле часу (хв), галочка після виконання

### Фаза 6 — Технічні вимоги

- [x] PWA Caching (офлайн-режим): застосунок і відмітки підходів зберігаються без інтернету; фонова синхронізація з Supabase коли з'являється зв'язок
- [x] Wake Lock API: екран телефону не блокується і не гасне під час активного тренування (вже є в плані, але потрібно реалізувати)
- [x] Вібрація + звук після таймера відпочинку — щоб не треба було дивитись в екран; тумблер вкл/викл у Профілі
- [x] Адаптація під різні розміри екранів телефонів (safe-area iPhone)
- [x] PWA manifest — щоб можна було додати на головний екран телефону як застосунок

## Нотатки

- Авторизація — Magic Link через Supabase Auth: захищає базу від публічного доступу без постійного введення паролю
- `personal_note` в таблиці `mf_exercises` замість окремої таблиці — простіший код, нотатка підтягується одразу з фото
- Оптимістичні оновлення для підходів — критично для залів у цоколі з поганим інтернетом
- `user_id` є в кожній таблиці — при масштабуванні нічого не переробляти
- Фото тренажерів — зберігати в Supabase Storage
- Спільна Supabase база з MintoFood — таблиці MintoFit мають префікс `mf_` щоб не плутатись
